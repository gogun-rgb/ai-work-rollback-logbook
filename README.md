# AI 작업 되돌리기 로그북

Codex, Claude Code, OpenCode가 수정한 파일을 확인하고 유지하거나 안전하게 되돌릴 수 있도록 돕는 초보자용 로컬 우선 웹 앱입니다.

## 프로젝트 소개

AI 코딩 도구가 여러 파일을 한꺼번에 바꿨을 때, 어떤 파일을 남기고 어떤 파일을 되돌릴지 기록하기 어렵습니다. 이 앱은 로컬 Git 프로젝트의 변경 사항을 읽고, 파일별 결정과 작업 메모를 SQLite에 저장합니다.

## 주요 기능

- 프로젝트 폴더 경로 검사
- Git 저장소 여부와 현재 브랜치 확인
- `git status` 기반 변경 파일 목록 표시
- 파일별 diff 또는 새 텍스트 파일 미리보기
- 파일별 유지, 되돌리기 예정, 미결정 기록
- AI 작업 요청, 실패 원인, 오류 메시지, 배운 점 저장
- 로그북 목록과 상세 조회
- 체크리스트 확인 후 선택한 추적 파일만 `git restore -- <file>`로 되돌리기
- 추적되지 않은 새 파일 자동 삭제 방지
- 상태가 다시 바뀐 파일의 되돌리기 차단

## 대상 사용자

- Codex 사용자
- Claude Code 사용자
- OpenCode 사용자
- Git과 터미널 사용이 아직 익숙하지 않은 초보 개발자
- AI가 바꾼 파일을 안전하게 검토하고 싶은 사용자

## 화면 구성

- 프로젝트: 프로젝트 폴더 경로를 입력하고 최근 프로젝트를 엽니다.
- 현재 변경 사항: 프로젝트를 다시 선택하고 변경 파일을 확인합니다.
- 로그북: 과거 작업 기록을 검색하고 상세 내용을 확인합니다.
- 사용 방법: Git, 변경 파일, 되돌리기 주의사항을 설명합니다.

## 프로젝트 폴더 경로 입력 방법

첫 화면의 입력창에 Git으로 관리 중인 프로젝트 폴더의 절대 경로를 넣습니다.

예시:

```text
C:\Example\sample-project
```

## Windows에서 경로 복사하는 방법

1. 파일 탐색기에서 프로젝트 폴더를 엽니다.
2. 주소 표시줄의 빈 곳을 클릭합니다.
3. 전체 경로를 복사합니다.
4. 앱의 프로젝트 경로 입력창에 붙여넣습니다.

## 안전한 되돌리기 원칙

- 폴더 전체를 한 번에 되돌리지 않습니다.
- 강제 초기화 명령을 사용하지 않습니다.
- 추적되지 않은 파일을 일괄 삭제하는 명령을 사용하지 않습니다.
- 사용자가 되돌리기 예정으로 선택한 파일만 처리합니다.
- 추적되지 않은 새 파일은 자동 삭제하지 않습니다.
- 되돌리기 직전에 Git 상태를 다시 확인합니다.
- 상태가 바뀐 파일은 되돌리기를 중단합니다.
- 프로젝트 폴더 밖의 파일은 조작하지 않습니다.

## 스크린샷 영역

아직 스크린샷은 포함하지 않았습니다. 첫 공개 이후 실제 화면을 캡처해 이 영역에 추가할 수 있습니다.

## MVP 제한 사항

- Git 저장소가 아닌 폴더는 등록할 수 없습니다.
- 추적되지 않은 새 파일은 앱에서 삭제하지 않습니다.
- 이름이 변경된 파일과 새로 추가되어 staged 된 파일의 자동 복원은 제한적으로 안내만 제공합니다.
- 여러 프로젝트를 한 화면에서 동시에 비교하지 않습니다.
- 원격 저장소 상태나 Pull Request 정보는 다루지 않습니다.

## 향후 개발 계획

1. 로그북 기록 내 diff 스냅샷 저장 옵션
2. 파일별 테스트 명령 메모
3. 스크린샷과 실행 결과 첨부
4. 복원 전 백업 파일 생성 옵션
5. 프로젝트별 기본 체크리스트 템플릿
