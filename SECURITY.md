# 보안 정책

이 앱은 로컬 파일과 Git 명령어를 다룹니다. 사용 전 중요한 프로젝트는 백업하거나 커밋을 만들어두세요.

## 취약점 제보

보안 취약점은 공개 Issue에 상세히 올리지 말아주세요. 재현에 필요한 최소 정보만 정리해 관리자가 안전하게 확인할 수 있는 경로로 전달해주세요.

## 올리면 안 되는 정보

- API 키
- 개인 데이터
- 인증 토큰
- 실제 프로젝트 경로
- 실제 작업 로그
- SQLite 데이터베이스 파일

## 안전 원칙

- 앱은 선택된 일부 추적 파일만 `git restore`로 되돌립니다.
- 앱은 추적되지 않은 새 파일을 자동 삭제하지 않습니다.
- 앱은 강제 초기화나 추적되지 않은 파일 일괄 삭제 명령을 사용하지 않습니다.
- 되돌리기 전에는 체크리스트를 모두 확인해야 합니다.
